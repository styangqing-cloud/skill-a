# Employee Interview Analyzer v2.2.0 — 全自动流程升级版

## 📌 项目概述

**项目名称：** employee-interview-analyzer（员工访谈结果智能分析技能）  
**版本升级：** v1.0.0 → v2.0.0 → v2.0.1 → v2.1.0 → v2.2.0  
**作者：** tommyyang@tencent  
**完成时间：** 2026-03-24  

### 升级目标

1. ✅ **增强方法论支撑**：从仅支持杨三角理论+假设驱动分析，升级为支持 7 种专业方法论
2. ✅ **智能场景识别**：自动识别访谈类型，无需用户手动指定
3. ✅ **方法论自动匹配**：根据场景自动推荐最优方法论组合
4. ✅ **与 skill-b 方法论完全对齐**：提纲生成和结果分析使用一致的方法论体系
5. ✅ **⭐ 场景询问确认**：每次分析前主动询问用户访谈场景，确保分析方向正确
6. ✅ **⭐ 方法论精简限制**：每场景最多3种方法论（1主+2辅），深度优先于广度
7. ✅ **📊 数据可视化**：支持雷达图、柱状图、折线图三种图表类型，直观展示分析结果
8. ✅ **👤 员工画像自动生成**：基于访谈内容生成员工画像，支持导出为MD/腾讯文档
9. ✅ **📝 访谈结论文章**：生成800-1500字科学合理的结论文章
10. ✅ **⭐⭐ 文本自动修正**：自动修正访谈文本中的错别字和语句不通的地方
11. ✅ **⭐⭐ 全自动流程**：去除所有询问步骤，自动判断并生成所有输出内容
12. ✅ **⭐ 战场画像**：员工画像中新增当前工作内容、表现、计划分析
13. ✅ **⭐ 组织画像**：组织诊断场景中自动生成组织健康度评估

## 🆕 v2.2.0 新增能力

### 1. 文本自动修正（Step 0.5）

| 特性 | 说明 |
|------|------|
| **错别字修正** | 常见同音字、形近字、输入法选词错误 |
| **语句不通修正** | 语序混乱、缺少主语宾语、重复冗余、标点错误 |
| **上下文理解修正** | 基于上下文推断正确用词，保留原意 |
| **修正记录** | 所有修正在附录中呈现，可追溯 |

**示例：**
- 原文："这个项目做的不错" → 修正后："这个项目做得不错"
- 原文："因为很多问题，所以导致" → 修正后："因为有很多问题，所以导致"

### 2. 数据可视化自动判断

| 条件 | 说明 | 触发可视化 |
|------|------|-----------|
| **多维度评分** | 3个以上维度的量化评分 | ✅ 自动生成 |
| **对比分析** | 多个对象/时间点/维度的对比 | ✅ 自动生成 |
| **趋势分析** | 多次访谈或时间序列数据 | ✅ 自动生成 |
| **优先级评估** | 问题评分、改进优先级等 | ✅ 自动生成 |
| **简单访谈** | 单次访谈、无量化评分、无对比 | ❌ 不生成 |

### 3. 员工画像自动呈现

**v2.2.0 变更：**
- ❌ 去除"是否需要导出员工画像"的询问
- ✅ 员工画像直接包含在完整分析报告中

**员工画像结构：**
1. 基础画像信息
2. 能力特征画像（专业能力、软技能、职业锚与价值观、AI素养）
3. 性格与行为模式
4. **⚔️ 战场画像**（v2.2.0 新增）
5. 发展潜力与风险

### 4. 战场画像（v2.2.0 新增）

**当前"战场"描述：**
- 核心工作内容、关键挑战、工作环境、战场特征

**当前"战场"表现（自我感知）：**
- 工作成果、自我评估、遇到的困难、优势领域

**后续工作计划：**
- 短期/中期/长期计划、能力提升需求、支持需求

**重要说明：** 如访谈材料中没有相关信息，此部分留空。

### 5. 组织画像（v2.2.0 新增，仅组织诊断场景）

**组织画像结构：**
1. 组织基础信息
2. 组织能力画像（杨三角理论）：员工思维、能力、治理三支柱
3. 组织健康度诊断（韦斯伯德六盒模型）：六维度评分和状态
4. 组织文化特征：文化类型、优势、挑战、一致性
5. 组织AI成熟度：5个维度评估
6. 组织优势与劣势
7. 组织改进优先级矩阵
8. 组织发展建议和变革路线图

### 6. 访谈结论文章自动生成

**v2.2.0 变更：**
- ❌ 去除"是否需要生成访谈结论文章"的询问
- ✅ 结论文章自动生成，直接包含在完整分析报告中

**文章规范：**
- 字数范围：800-1500字
- 科学合理、易读性强、切合访谈事实

### 7. 多访谈材料处理优化

**处理规则：**
1. 逐份修正访谈文本
2. 分别生成每人的员工画像（含战场画像）
3. 交叉分析，找共性和差异
4. 自动生成综合组织画像（组织诊断场景）
5. 自动生成综合结论文章

**输出规则：**
- 如有多个员工，按访谈顺序依次呈现
- 每个员工的独立画像包含完整内容

## 🆕 v2.1.0 新增能力

### 1. 智能场景识别引擎

| 特性 | 说明 |
|------|------|
| **三层识别** | 关键词匹配(30%) + 语义分析(45%) + 上下文推断(25%) |
| **10种场景** | 入职/绩效/离职/晋升/职业发展/团队反馈/挽留/转岗/组织诊断/招聘面试 |
| **子类型** | 离职(主动/被动)、组织诊断(快速/全面)、面试(技术/综合) |
| **置信度** | 高(≥0.8自动)/中(0.6-0.8确认)/低(<0.6引导) |
| **AI信号** | 自动检测AI相关话题，激活AI增强分析维度 |

### 2. 方法论自动匹配引擎

| 特性 | 说明 |
|------|------|
| **主+辅组合** | 每个场景有主方法论+最多2个辅助方法论 |
| **⭐ 数量硬限制** | 每场景 ≤ 3种方法论（1主+最多2辅），确保深度优先 |
| **动态调整** | 根据访谈内容动态调整辅助方法论（达上限时替换而非追加） |
| **用户覆盖** | 支持用户手动指定/更换方法论（同受 ≤3种限制） |
| **多方法论交叉** | 在3种方法论范围内做交叉验证 |

### 3. 七大方法论分析框架

| 编号 | 方法论 | 分析场景 | 核心分析能力 |
|-----|--------|---------|-------------|
| M1 | **杨三角理论** ⭐ | 组织诊断 | 思维/能力/治理三支柱评估 + 短板追因 + 归因分析 |
| M2 | **韦斯伯德六盒** | 组织诊断 | 六维度评估 + 系统性vs个人问题鉴别 + 优先级矩阵 |
| M3 | **推拉力理论** | 离职/挽留 | 四力提取 + 强度评估 + 挽留可能性矩阵 + 挽留方案 |
| M4 | **冰山模型** | 人才评估 | 六层证据提取 + 水面以下深挖 + 岗位匹配评估 |
| M5 | **BEM 模型** | 绩效分析 | 先环境后个人 + 六因素分析 + 根因排序 + 改进方案 |
| M6 | **职业锚+GROW** | 职业发展 | 职业锚识别 + GROW四阶段评估 + 发展路径建议 |
| M7 | **STAR 面试法** | 面试评估 | STAR完整度评分 + 红旗检测 + 能力评估 + 录用建议 |

### 4. AI 时代增强分析维度

| 维度 | 分析焦点 |
|------|---------|
| 🛠️ AI 工具素养 | 员工使用AI工具的广度/深度 |
| 🔄 人机协作 | 员工与AI协作的成熟度 |
| 🔮 岗位演进认知 | 员工对AI影响岗位的理解 |
| 📈 能力模型更新 | 员工的能力进化意愿和行动 |
| 🏢 组织AI成熟度 | 组织AI支持环境的评估 |

### 5. 深度分析方法论

| 方法 | 说明 |
|------|------|
| **假设驱动** | 先形成假设，用证据验证/推翻 |
| **矛盾检测** | 自动识别言行矛盾、前后矛盾、社会期望偏差 |
| **5 Whys追因** | 对关键发现逐层追问5次至根本原因 |
| **交叉验证** | 多方法论交叉验证，提升分析可信度 |

### 6. 结构化报告模板

| 模板 | 用途 |
|------|------|
| **通用模板** | 适用于所有场景的标准分析报告 |

## 🚀 使用方式

> 💡 **v2.2.0 新特性：** 全自动流程，去除所有询问步骤，自动判断并生成所有输出内容。

### 最简用法
```
"帮我分析这份访谈记录：[粘贴访谈内容]"
```
→ 分析器会先询问场景，然后：
  1. 自动修正文本中的错别字和语句不通
  2. 自动匹配方法论（≤3种）并执行分析
  3. 自动生成员工画像（含战场画像）
  4. 自动判断是否需要生成数据可视化
  5. 自动生成访谈结论文章
  6. 输出完整的结构化分析报告

### 指定场景
```
"这是一份绩效访谈记录，帮我分析绩效下滑的根因：[粘贴内容]"
```
→ 分析器简要确认场景后直接开始全自动分析流程

### 指定方法论（最多3种）
```
"用推拉力理论分析这份离职访谈：[粘贴内容]"
```
→ 按指定方法论执行分析，同时自动生成所有输出

### 多方法论交叉（≤3种）
```
"同时用杨三角和韦斯伯德分析这份组织诊断：[粘贴内容]"
```
→ 执行交叉分析，自动生成组织画像等所有输出

### 多文件综合
```
"这5份组织诊断访谈，帮我做综合分析：[上传/粘贴多份记录]"
```
→ 分别分析每份访谈，生成各自的员工画像，再综合生成组织画像
