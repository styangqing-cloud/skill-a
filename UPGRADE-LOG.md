### ⭐⭐ 新增功能一：文本自动修正（Step 0.5）

**功能描述：**
在开始正式分析前，先根据上下文修正访谈记录中的错别字和语句不通顺的地方，确保分析基于准确清晰的文本。

**修正范围：**
1. **错别字修正**
   - 常见同音字错误（如：的/得/地、在/再、已/以等）
   - 形近字错误
   - 输入法选词错误

2. **语句不通修正**
   - 语序混乱的句子
   - 缺少主语或宾语的句子
   - 重复冗余的表达
   - 标点符号错误

3. **上下文理解修正**
   - 基于上下文推断可能的正确用词
   - 保留口语化表达，但修正明显错误
   - 不改变原意，仅提升可读性

**修正规则示例：**

| 规则类型 | 示例 | 修正后 |
|---------|------|-------|
| 错别字 | "这个项目做的不错" | "这个项目做得不错" |
| 语句不通 | "因为很多问题，所以导致" | "因为有很多问题，所以导致" |
| 标点错误 | "我们要解决问题，然后才能前进" | "我们要解决问题，然后才能前进" |

**修正输出：**
- 所有修正记录在分析报告附录中呈现
- 包含原文、修正后、修正理由
- 确保修正过程可追溯

**技术实现：**
```python
def correct_interview_text(text: str) -> dict:
    """
    修正访谈文本中的错误
    返回: {
        "corrected_text": "修正后的文本",
        "corrections": [
            {
                "original": "原文片段",
                "corrected": "修正后片段",
                "type": "错别字/语句不通/标点错误",
                "reason": "修正理由"
            }
        ]
    }
    """
    # 1. 错别字修正
    # 2. 语句不通修正
    # 3. 标点符号修正
    # 4. 上下文一致性检查
```

**注意事项：**
- ⚠️ 保留原汁原味：口语化表达不必过度修正
- ⚠️ 避免误判：不确定的用词不要随意修改
- ⚠️ 记录修正：所有修正都要记录在报告中
- ✅ 提升准确性：文本修正有助于提升后续分析准确性

### ⭐⭐ 新增功能二：数据可视化自动判断

**功能描述：**
根据访谈内容自动判断是否需要生成数据可视化图表，无需询问用户。

**自动判断规则：**

| 条件 | 说明 | 触发可视化 |
|------|------|-----------|
| **多维度评分** | 访谈分析中有3个以上维度的量化评分 | ✅ 自动生成 |
| **对比分析** | 存在多个对象、多个时间点、多个维度的对比 | ✅ 自动生成 |
| **趋势分析** | 多次访谈或时间序列数据 | ✅ 自动生成 |
| **优先级评估** | 问题评分、改进优先级等 | ✅ 自动生成 |
| **简单访谈** | 单次访谈、无量化评分、无对比需求 | ❌ 不生成 |

**判断逻辑：**
```python
def should_generate_visualization(analysis_result: dict) -> dict:
    """
    判断是否需要生成数据可视化
    """
    chart_types = []
    
    # 1. 检查是否有量化评分（3个以上维度）
    if len(analysis_result.get("scores", {})) >= 3:
        chart_types.append("radar")
    
    # 2. 检查是否有对比数据
    if analysis_result.get("has_comparison"):
        chart_types.append("bar")
    
    # 3. 检查是否有时间序列
    if analysis_result.get("has_trend"):
        chart_types.append("line")
    
    # 4. 决定是否生成
    should_generate = len(chart_types) > 0
    
    return {
        "should_generate": should_generate,
        "chart_types": chart_types,
        "reason": "检测到{}类适合可视化的数据".format(len(chart_types))
    }
```

**支持的可视化类型：**
1. **雷达图**：多维度综合评估（杨三角、韦斯伯德、冰山模型等）
2. **柱状图**：维度对比分析（健康度评分、选项统计等）
3. **折线图**：趋势分析（时间序列、多次访谈对比等）

### ⭐ 新增功能四：战场画像（员工画像子模块）

**功能描述：**
"战场画像"描述员工当前的工作内容、表现和计划，是员工画像的重要组成部分。

**战场画像结构：**

#### 1. 当前"战场"描述
- **核心工作内容**：工作领域1、2、3
- **关键挑战**：挑战1、2
- **工作环境**：环境描述
- **战场特征**：复杂度、不确定性、压力水平、技术前沿性

#### 2. 当前"战场"表现（自我感知）
- **工作成果**：成果1、2 + 证据
- **自我评估**：工作满意度、能力发挥度、成就感来源
- **遇到的困难**：困难1、2 + 应对方式
- **优势领域**：优势1、2 + 表现

#### 3. 后续工作计划
- **短期计划（1-3个月）**：计划1、2
- **中期计划（3-6个月）**：计划1、2
- **长期展望（6个月以上）**：展望1、2
- **能力提升需求**：需提升能力1、2 + 提升计划
- **支持需求**：支持需求1、2

**重要说明：**
- 如访谈材料中没有相关信息，此部分留空
- 基于访谈内容提取，不编造信息
- 保留员工的自我感知，不主观评判

### 📝 新增功能三：访谈结论文章生成

**文章规范：**
- **字数范围**：800-1500字
- **科学合理**：基于访谈事实，避免主观臆断
- **易读性强**：结构清晰、语言流畅、重点突出
- **切合事实**：引用访谈内容作为支撑证据

**文章结构（标准模板）：**
```markdown
1. 访谈背景（150字）
   - 访谈时间、对象、目的
   - 访谈场景概述
   - 核心议题

2. 核心发现（400字）
   - 主要发现1 + 证据
   - 主要发现2 + 证据
   - 主要发现3 + 证据

3. 深度分析（300字）
   - 问题根因分析
   - 矛盾点分析
   - 关键洞察

4. 建议与展望（250字）
   - 短期建议
   - 长期展望
   - 后续行动

5. 总结（100字）
   - 核心结论
   - 关键要点
```

1. **组织诊断型**：适用于组织诊断访谈
2. **绩效评估型**：适用于绩效访谈
3. **离职分析型**：适用于离职访谈
4. **职业发展型**：适用于职业发展访谈
5. **自动推荐**：根据访谈内容自动推荐类型

| 维度 | 评估标准 | 满分 |
|------|---------|------|
| 科学性 | 基于事实、论证合理、方法科学 | 30 |
| 准确性 | 数据准确、引用正确、无误导性表述 | 25 |

