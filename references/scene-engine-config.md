# 智能场景识别与方法论自动匹配引擎

## 📖 引擎概述

本文档定义了访谈分析技能的**核心智能引擎**：如何从访谈文本中自动识别场景类型，并匹配最优的方法论组合进行分析。

---

## 🧠 场景识别引擎

### 三层识别机制

```
┌───────────────────────────────────────────────────┐
│              智能场景识别引擎                       │
├───────────────────────────────────────────────────┤
│                                                    │
│  第一层：关键词匹配（权重 30%）                     │
│  ├─ 一级关键词（强信号）× 3                        │
│  └─ 二级关键词（弱信号）× 1                        │
│                                                    │
│  第二层：语义分析（权重 45%）                       │
│  ├─ 话题聚类分析                                   │
│  ├─ 情感基调分析                                   │
│  └─ 讨论结构分析                                   │
│                                                    │
│  第三层：上下文推断（权重 25%）                     │
│  ├─ 参与者身份推断                                 │
│  ├─ 时间节点推断                                   │
│  └─ 对话动态推断                                   │
│                                                    │
│  → 输出：场景类型 + 置信度 + 子类型                │
└───────────────────────────────────────────────────┘
```

### 语义特征识别规则

| 场景 | 情感基调 | 话题特征 | 对话结构 |
|------|---------|---------|---------|
| **入职访谈** | 积极/好奇/焦虑 | 环境、团队、流程、期望 | 引导式、探索性 |
| **绩效访谈** | 混合/紧张/反思 | 成果、目标、差距、改进 | 评估式、反馈式 |
| **离职访谈** | 消极/释然/复杂 | 原因、建议、感受、告别 | 开放式、收集式 |
| **晋升访谈** | 紧张/期待/焦虑 | 标准、差距、能力、计划 | 评估式、发展式 |
| **职业发展** | 积极/迷茫/期待 | 方向、目标、路径、资源 | 教练式、探索式 |
| **团队反馈** | 混合/坦诚/顾虑 | 协作、管理、文化、问题 | 收集式、匿名感 |
| **挽留访谈** | 犹豫/纠结/感动 | 条件、诚意、比较、决策 | 说服式、谈判式 |
| **转岗访谈** | 积极/好奇/不确定 | 新方向、匹配、挑战、风险 | 评估式、探索式 |
| **组织诊断** | 反思/系统/分析 | 制度、流程、能力、文化 | 系统式、诊断式 |
| **招聘面试** | 紧张/展示/自信 | 经历、能力、项目、动机 | 评估式、追问式 |

### 动态调整规则

在基础映射之上，引擎会根据访谈内容进行动态调整：

> **⚠️ 动态调整约束：** 当触发新增方法论时，如果当前已达到3种上限，必须**替换**优先级最低的辅助方法论，而非无限追加。

| 条件 | 调整动作 | 理由 | 替换规则 |
|------|---------|------|---------|
| 绩效访谈中发现大量组织因素 | 增加 M1(杨三角)为辅助 | 绩效问题可能有组织根因 | 如已满3种，替换优先级最低的辅助 |
| 离职访谈中员工表达职业迷茫 | 增加 M6(职业锚)为辅助 | 职业方向不匹配可能是深层推力 | 如已满3种，替换优先级最低的辅助 |
| 入职访谈中发现岗位期望落差 | 增加 M4(冰山)为辅助 | 评估入职前的岗位认知匹配 | 如已满3种，替换优先级最低的辅助 |
| 晋升访谈中发现环境障碍 | 增加 M5(BEM)为辅助 | 晋升阻碍可能来自环境而非个人 | 如已满3种，替换优先级最低的辅助 |
| 任何场景中出现 AI 相关话题 | 激活 AI 增强分析维度 | AI 维度需要特殊关注 | AI维度独立于方法论计数 |
| 单次访谈涉及多个场景 | 启用多方法论组合分析 | 复杂访谈需要多维度覆盖 | 仍受 ≤3种方法论限制 |

### 置信度计算

```
置信度 = (关键词得分 × 0.30) + (语义得分 × 0.45) + (上下文得分 × 0.25)

判定阈值：
- ≥ 0.80：高置信度 → 自动执行分析
- 0.60-0.80：中置信度 → 展示推荐并确认
- < 0.60：低置信度 → 引导用户指定场景
```

## ⚠️ 引擎使用注意事项

1. **场景询问是第一步**：每次分析前必须先主动询问用户确认访谈场景（Step 0），即使智能识别已有高置信度结果
2. **方法论数量硬限制**：每个场景最多使用3种方法论（1主+最多2辅），动态调整和用户指定均受此限制
3. **场景识别是手段不是目的**：识别场景是为了选择最优分析框架，用户可随时覆盖
4. **多场景共存**：一次访谈可能涉及多个场景，引擎支持多场景组合分析（但方法论总数仍 ≤ 3）
5. **方法论不是教条**：方法论是分析工具而非限制，灵活运用更重要
6. **持续迭代**：随着使用场景积累，关键词库和匹配规则应持续优化
7. **用户反馈闭环**：记录用户对自动匹配结果的接受/覆盖情况，优化引擎
