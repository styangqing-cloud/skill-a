# 杨三角理论 - 访谈结果分析指南

## 📖 分析定位

本指南用于**分析访谈结果**中与组织能力相关的内容，通过杨三角理论的三大支柱进行系统性诊断。

> **与提纲生成器（skill-b）的区别：** skill-b 用杨三角设计提问，本指南用杨三角分析回答。

---

## 🎯 分析框架

### 三大支柱分析模型

```
                    员工思维 (Mindset)
                   愿不愿意？
                      ▲
                     / \
                    / ⭐ \
                   /       \
                  /  组织    \
                 /   能力     \
                /             \
员工能力 ◄───────────────────────► 员工治理
(Competence)                     (Governance)
 会不会？                          允不允许？
```

### 分析步骤

#### Step 1: 内容分类 — 将访谈内容归入三大支柱

**分类规则：**

| 支柱 | 关键词/语义信号 | 示例陈述 |
|------|---------------|---------|
| **员工思维** | 想/不想、认同/不认同、意义/价值、愿意/不愿意、热情/倦怠 | "我觉得这个方向没什么意义"、"大家都不太buy-in这个战略" |
| **员工能力** | 会/不会、能/不能、学习/培训、技能/经验、差距/不足 | "团队缺乏AI工程化能力"、"新人上手太慢" |
| **员工治理** | 可以/不可以、允许/不允许、流程/制度、资源/工具、权限/审批 | "审批流程太长了"、"资源总是申请不到" |

**分类注意事项：**
- 一条陈述可能涉及多个支柱，需要判断**主要归属**
- 注意区分「个人层面」和「组织层面」的陈述
- 情绪性表达要还原到具体支柱

#### Step 2: 支柱健康度评估

**评估维度（每个支柱）：**

| 评估维度 | 1分（严重不足） | 3分（有改进空间） | 5分（运作良好） |
|---------|--------------|-----------------|--------------|
| **证据数量** | 大量负面证据 | 正负参半 | 以正面为主 |
| **问题严重度** | 影响核心运作 | 局部影响 | 仅边缘问题 |
| **改进可行度** | 需要大幅变革 | 可逐步改进 | 微调即可 |
| **员工感知** | 普遍消极 | 认知分化 | 普遍积极 |

**输出模板：**

```
支柱健康度评分卡：
┌───────────┬───────┬───────────────────────────────┐
│   支柱     │ 评分  │ 关键证据                       │
├───────────┼───────┼───────────────────────────────┤
│ 员工思维   │ X/5   │ • 证据1                       │
│           │       │ • 证据2                       │
├───────────┼───────┼───────────────────────────────┤
│ 员工能力   │ X/5   │ • 证据1                       │
│           │       │ • 证据2                       │
├───────────┼───────┼───────────────────────────────┤
│ 员工治理   │ X/5   │ • 证据1                       │
│           │       │ • 证据2                       │
└───────────┴───────┴───────────────────────────────┘
短板支柱：[最低分的支柱]
```

#### Step 3: 短板支柱深度分析

**分析原则：木桶效应 — 组织能力取决于最短的那块板**

对评分最低的支柱进行 5 Whys 追因：

```
短板支柱: 员工治理 (2/5)
  Why 1: 为什么治理得分低？→ 流程冗长，审批效率低
  Why 2: 为什么审批效率低？→ 决策权过度集中在上层
  Why 3: 为什么决策权集中？→ 管理层对下放权力不放心
  Why 4: 为什么不放心？→ 缺乏有效的过程监控机制
  Why 5: 为什么缺乏监控？→ 数字化管理工具不足
  
  根因: 治理基础设施（数字化工具+监控机制）不足
  → 导致管理层不敢授权 → 导致流程冗长
```

#### Step 4: 交叉影响分析

**支柱间因果关系识别：**

| 因 → 果 | 典型模式 | 识别信号 |
|---------|---------|---------|
| 治理 → 思维 | 制度不合理导致员工消极 | "制度这样我也没什么动力" |
| 治理 → 能力 | 资源不足导致能力无法发挥 | "想学但没时间/资源" |
| 能力 → 思维 | 能力不足导致信心丧失 | "感觉自己跟不上了" |
| 思维 → 能力 | 消极心态导致不愿学习 | "学了也没用" |
| 能力 → 治理 | 能力不足导致过度管控 | "他们不行所以我得盯着" |
| 思维 → 治理 | 不认同导致抵制制度 | "这个流程没意义我不想走" |

#### Step 5: 杨三角归因分析

对每个关键发现进行归因：

| 归因类型 | 核心问题 | 干预方向 |
|---------|---------|---------|
| **思维归因** | 员工的价值观/态度/认同出了问题 | 文化建设、沟通对齐、激励重塑 |
| **能力归因** | 员工的知识/技能/能力不够 | 培训发展、招聘补充、团队学习 |
| **治理归因** | 组织的制度/流程/资源有问题 | 流程优化、资源配置、授权机制 |

## ⚠️ 分析注意事项

1. **避免预设结论**：不要带着"一定是治理问题"的假设去分析
2. **多人验证**：单个人的访谈只代表个体视角，组织诊断需多人交叉验证
3. **区分层级**：高管/中层/一线看到的支柱问题可能完全不同
4. **动态视角**：支柱健康度是动态的，关注趋势而非静态评分
5. **行动导向**：分析的目的是产出可执行的改进建议，避免"诊断完就完了"
