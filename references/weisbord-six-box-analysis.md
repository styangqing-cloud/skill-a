# 韦斯伯德六盒模型 - 访谈结果分析指南

## 📖 分析定位

本指南用于**分析访谈结果**中的组织健康度问题，通过六个维度系统性评估组织运作状态，重点区分「系统性问题」和「个人抱怨」。

---

## 🎯 分析框架

### 六维度分析模型

```
┌─────────────────────────────────────────┐
│              外部环境                    │
├─────────────┬─────────────┬─────────────┤
│   ① 目的    │   ② 结构    │   ③ 关系    │
│  目标清晰？  │  分工合理？  │  协作顺畅？ │
├─────────────┼─────────────┼─────────────┤
│   ④ 奖励    │   ⑤ 领导    │  ⑥ 支持机制 │
│  激励有效？  │  方向明确？  │  工具到位？ │
└─────────────┴─────────────┴─────────────┘
```

### 分析步骤

#### Step 1: 内容归维度

将访谈中的每条陈述归类到对应维度：

| 维度 | 归类关键词/信号 | 典型陈述 |
|------|---------------|---------|
| **目的** | 目标、方向、战略、使命、愿景 | "不知道团队今年的方向"、"目标变来变去" |
| **结构** | 分工、职责、层级、流程、组织架构 | "不知道该找谁负责"、"汇报线太混乱" |
| **关系** | 协作、沟通、冲突、信任、部门墙 | "跨部门合作太难了"、"信息不透明" |
| **奖励** | 薪酬、晋升、认可、激励、公平 | "做多做少一样"、"晋升标准不清楚" |
| **领导** | 管理、决策、方向、赋能、信任 | "领导朝令夕改"、"没有决策的权力" |
| **支持机制** | 工具、系统、培训、资源、AI工具 | "系统太难用了"、"缺少好用的工具" |

#### Step 2: 系统性问题 vs 个人抱怨 鉴别

**这是韦斯伯德分析的核心价值 — 区分系统问题和个人问题**

| 鉴别维度 | 系统性问题 | 个人抱怨 |
|---------|-----------|---------|
| **波及范围** | 多人反映/影响面广 | 仅个人感受 |
| **持续时间** | 长期存在/反复出现 | 偶发/短期 |
| **结构归因** | 制度/流程/机制层面 | 个人能力/态度/期望 |
| **可复现性** | 换人也会遇到 | 换人可能不会 |
| **解决路径** | 需要制度/流程变革 | 需要个人调整/沟通 |

**鉴别示例：**

| 陈述 | 判定 | 理由 |
|------|------|------|
| "目标传达不清楚" | 需进一步判断 | 如果只有一人说→可能是个人沟通；多人说→系统性 |
| "审批流程要走5个人" | 系统性问题 | 流程设计问题，影响所有人 |
| "我和直属leader性格不合" | 个人抱怨 | 人际层面，非制度问题 |
| "晋升标准年年变" | 系统性问题 | 制度稳定性问题 |

#### Step 3: 维度评分

对每个维度打分（1-5分），基于：

| 分值 | 含义 | 判断标准 |
|------|------|---------|
| 5 | 运作良好 | 无系统性问题，正面评价为主 |
| 4 | 基本良好 | 有小问题但不影响运作 |
| 3 | 有改进空间 | 存在系统性问题但可控 |
| 2 | 问题明显 | 系统性问题影响运作效率 |
| 1 | 严重问题 | 系统性障碍，严重影响组织效能 |

#### Step 4: 维度关联分析

分析维度之间的因果链：

```
常见关联模式：
目的不清 → 结构混乱 → 关系紧张 → 领导失效
奖励失调 → 关系恶化 → 协作障碍
支持缺失 → 效率低下 → 目的难达成
领导缺位 → 目的不清 + 结构混乱 + 奖励失调
```

#### Step 5: 问题优先级矩阵

```
           影响范围
        小           大
频  高 │ 快速修复  │ 🔴 紧急  │
率     │   ②      │  优先 ①  │
    ───┼──────────┼─────────┤
    低 │ 观察监控  │ 规划    │
       │   ④      │ 解决 ③  │
       └──────────┴─────────┘
```

## ⚠️ 分析注意事项

1. **不要急于归类**：有些陈述可能跨越多个维度，需要综合判断
2. **系统性判定需慎重**：单人访谈无法充分判定系统性，多人交叉才可靠
3. **关注沉默维度**：如果某个维度几乎没有陈述，可能是「被回避」或「无感知」
4. **领导维度敏感**：员工可能因顾虑而不直接评论领导，需要间接推断
5. **支持机制易被忽略**：很多问题的根因在支持机制，但表面看像其他维度
