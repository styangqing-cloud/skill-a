# 吉尔伯特 BEM 模型 - 访谈结果分析指南

## 📖 分析定位

本指南用于**分析绩效相关访谈结果**，通过 BEM 模型的六因素框架找到绩效差距的根本原因，核心原则是「**先环境后个人**」。

---

## 🎯 分析框架

### 六因素分析模型

```
┌─────────────────────────────────────────────────────┐
│              影响绩效的六大因素                        │
├─────────────────────┬───────────────────────────────┤
│  环境因素 (75%) ⭐   │    个人因素 (25%)             │
│  【先分析这边！】     │    【后分析这边！】           │
├─────────────────────┼───────────────────────────────┤
│  ① 信息与反馈       │   ④ 知识与技能                │
│  目标清晰？反馈及时？│   具备所需知识技能？          │
├─────────────────────┼───────────────────────────────┤
│  ② 资源与工具       │   ⑤ 天赋与能力                │
│  工具到位？资源充足？│   天赋匹配？潜力如何？        │
├─────────────────────┼───────────────────────────────┤
│  ③ 激励与后果       │   ⑥ 动机与态度                │
│  激励有效？后果明确？│   有内在动机？态度积极？      │
└─────────────────────┴───────────────────────────────┘
```

> **核心原则**：绩效问题的 75% 可归因于环境因素。**必须先排查环境因素**，避免「培训万能论」。

### 分析步骤

#### Step 1: 明确绩效差距

| 维度 | 内容 |
|------|------|
| **期望绩效** | 岗位/角色应该达到的标准 |
| **实际绩效** | 员工实际的表现水平 |
| **差距描述** | 具体差距的表现（量化优先） |
| **差距影响** | 差距对业务/团队的影响 |

#### Step 2: 环境因素分析（先！权重 75%）

**① 信息与反馈**

| 分析点 | 从访谈中提取 | 评估 |
|--------|-------------|------|
| 目标清晰度 | 员工是否明确知道期望目标？ | 🟢/🟡/🔴 |
| 标准明确度 | 员工是否知道"做好"的标准？ | 🟢/🟡/🔴 |
| 反馈及时性 | 员工是否获得及时的绩效反馈？ | 🟢/🟡/🔴 |
| 反馈质量 | 反馈是否具体可操作？ | 🟢/🟡/🔴 |
| 信息获取 | 员工能否获取所需的工作信息？ | 🟢/🟡/🔴 |

**常见问题信号：**
- "不知道leader到底想要什么" → 目标传达不清
- "做完了没人说好不好" → 缺乏反馈
- "标准一直在变" → 标准不稳定

**② 资源与工具**

| 分析点 | 从访谈中提取 | 评估 |
|--------|-------------|------|
| 工具支持 | 工作工具是否满足需求？ | 🟢/🟡/🔴 |
| 时间资源 | 是否有足够时间完成工作？ | 🟢/🟡/🔴 |
| 人力资源 | 人手是否充足？ | 🟢/🟡/🔴 |
| 预算资源 | 预算是否充足？ | 🟢/🟡/🔴 |
| AI 工具 | AI 工具是否可用/好用？ | 🟢/🟡/🔴 |
| 技术环境 | 开发/运维环境是否支持？ | 🟢/🟡/🔴 |

**常见问题信号：**
- "系统太卡了" → 工具问题
- "人手不够" → 人力不足
- "没有预算做这个" → 资源受限

**③ 激励与后果**

| 分析点 | 从访谈中提取 | 评估 |
|--------|-------------|------|
| 薪酬激励 | 薪酬是否与绩效挂钩？ | 🟢/🟡/🔴 |
| 晋升通道 | 是否有清晰的晋升路径？ | 🟢/🟡/🔴 |
| 认可机制 | 好的表现是否被认可？ | 🟢/🟡/🔴 |
| 后果明确 | 不良表现是否有明确后果？ | 🟢/🟡/🔴 |
| 公平感 | 员工是否感到公平？ | 🟢/🟡/🔴 |

**常见问题信号：**
- "做多做少一样" → 激励失效
- "干得好也没人说" → 认可缺失
- "他那样也没事" → 后果不明确

#### Step 3: 个人因素分析（后！权重 25%）

> ⚠️ **只有在环境因素排查完毕后**，才进入个人因素分析。如果环境因素有重大问题，应先解决环境问题。

**④ 知识与技能**

| 分析点 | 从访谈中提取 | 评估 |
|--------|-------------|------|
| 专业知识 | 是否具备岗位所需专业知识？ | 🟢/🟡/🔴 |
| 技术技能 | 是否具备所需技术技能？ | 🟢/🟡/🔴 |
| 经验积累 | 是否有足够的相关经验？ | 🟢/🟡/🔴 |
| 学习能力 | 是否能快速学习新知识？ | 🟢/🟡/🔴 |

**⑤ 天赋与能力**

| 分析点 | 从访谈中提取 | 评估 |
|--------|-------------|------|
| 认知能力 | 逻辑思维/分析能力 | 🟢/🟡/🔴 |
| 沟通能力 | 表达/倾听/协调能力 | 🟢/🟡/🔴 |
| 领导力 | 影响力/决策力（如需） | 🟢/🟡/🔴 |
| 抗压能力 | 压力下的表现 | 🟢/🟡/🔴 |

**⑥ 动机与态度**

| 分析点 | 从访谈中提取 | 评估 |
|--------|-------------|------|
| 内在动机 | 对工作内容的兴趣 | 🟢/🟡/🔴 |
| 职业承诺 | 对职业发展的投入度 | 🟢/🟡/🔴 |
| 主动性 | 是否主动寻求改进 | 🟢/🟡/🔴 |
| 责任心 | 对工作质量的重视度 | 🟢/🟡/🔴 |

#### Step 4: 根因排序

将所有因素按影响程度排序：

```
绩效差距根因排序：
┌──────┬──────────────────────┬──────┬──────────────┐
│ 排名 │ 根因                  │ 类型 │ 影响权重      │
├──────┼──────────────────────┼──────┼──────────────┤
│  1   │ 目标不清晰/变动频繁   │ 环境 │ ★★★★★     │
│  2   │ 人手不足/工作超负荷   │ 环境 │ ★★★★      │
│  3   │ 缺乏必要的技术技能   │ 个人 │ ★★★        │
│  4   │ 激励机制失效         │ 环境 │ ★★★        │
│  5   │ 内在动机下降         │ 个人 │ ★★          │
└──────┴──────────────────────┴──────┴──────────────┘
```

#### Step 5: 改进方案

| 时间 | 改进方向 | 具体措施 | 责任方 |
|------|---------|---------|-------|
| **短期（1-2周）** | 环境优化 | 澄清目标/补充资源 | 管理者 |
| **中期（1-3月）** | 能力提升 | 培训/教练/实践 | HR + 管理者 |
| **长期（3-12月）** | 系统改进 | 制度优化/流程再造 | 组织 |

## ⚠️ 分析注意事项

1. **先环境后个人**：这是 BEM 的核心原则，不要跳过环境直接归咎个人
2. **避免培训万能论**：如果根因是环境因素，培训不能解决问题
3. **环境因素相互关联**：信息不清→资源浪费→激励失效，要看整体
4. **个人因素中动机最关键**：知识和技能可以培养，但动机问题更深层
5. **量化差距**：尽量用数据描述绩效差距，避免模糊的定性描述
