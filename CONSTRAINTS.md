## ⭐⭐⭐ 方法论数量强制约束

### 核心规则

**每次访谈分析严格限制最多使用 3 种方法论**

- **主方法论**: 1个（必选）
- **辅助方法论**: 最多2个
- **总计**: 不超过3种方法论

### 约束性质

**⚠️ 此为强制约束，任何情况下都不可违背**

- ✅ **自动匹配**: 技能自动匹配方法论时必须遵守
- ✅ **用户指定**: 即使用户要求使用更多方法论，也必须拒绝并遵守限制
- ✅ **场景识别**: 无论访谈场景如何复杂，都不能突破此限制
- ✅ **分析深度**: 宁愿深入应用2-3个方法论，也不浅尝辄止地应用多个

### 约束理由

1. **深度优先**: 深入应用少数方法论比浅层应用多个方法论更有价值
2. **分析聚焦**: 避免分析过于发散，确保洞察聚焦
3. **质量保证**: 方法论过多会导致分析维度分散，影响洞察质量
4. **用户理解**: 3个方法论是用户理解消化能力的合理上限

### 强制执行机制

#### 代码层面

```python
# 硬限制常量（不可修改）
MAX_METHODOLOGIES = 3  # ⭐⭐⭐ 强制硬限制

def validate_methodology_count(methodologies: list) -> bool:
    """验证方法论数量是否符合强制约束"""
    if len(methodologies) > MAX_METHODOLOGIES:
        raise ValueError(
            f"方法论数量超过强制限制！当前：{len(methodologies)}，"
            f"最大允许：{MAX_METHODOLOGIES}。这是不可违背的约束。"
        )
    return True
```

#### 自动匹配层面

```python
def match_methodology(scene: str, transcript: str) -> dict:
    """自动匹配方法论 - 必须遵守强制约束"""
    primary = get_primary_methodology(scene)
    secondary_candidates = get_secondary_candidates(scene, transcript)
    
    # ⭐⭐⭐ 强制限制：辅助方法论最多2个
    MAX_SECONDARY = MAX_METHODOLOGIES - 1  # = 2
    
    # 按优先级排序，保留前2个
    secondary = sorted(
        secondary_candidates, 
        key=lambda m: m.priority, 
        reverse=True
    )[:MAX_SECONDARY]
    
    return {
        "primary": primary,
        "secondary": secondary,
        "total": 1 + len(secondary),  # 必须保证 ≤ 3
        "constraint_complied": True
    }
```

### 违约后果

**违反此约束将被视为技能实现错误**

- ❌ 分析报告质量下降
- ❌ 用户理解成本增加
- ❌ 洞察聚焦度降低
- ❌ 技能信誉受损

### 合规检查清单

在生成分析报告前，必须检查：

- [ ] 方法论总数是否 ≤ 3
- [ ] 主方法论是否为1个
- [ ] 辅助方法论是否 ≤ 2个
- [ ] 是否已向用户说明此约束（如果用户指定）
- [ ] 是否已拒绝用户不合理的方法论数量要求

### 例外情况

**⚠️ 无例外情况**

此约束适用于所有场景、所有情况、所有用户请求，没有任何例外。

## 其他核心约束

### 1. 场景询问约束

**每次分析前必须主动询问用户访谈场景**

- 即使智能识别置信度很高
- 即使用户提供了明确的访谈内容
- 必须向用户确认场景类型

### 2. 文本修正约束

**必须在分析前自动修正访谈文本**

- 修正错别字
- 修正语句不通
- 记录所有修正内容

### 3. 画像生成约束

**员工画像必须自动生成，无需询问用户**

- 必须包含战场画像（如有信息）
- 必须包含能力、性格、价值观等完整画像
- 组织诊断时必须生成组织画像

### 4. 可视化判断约束

**数据可视化必须自动判断，无需询问用户**

- 基于分析结果智能判断
- 雷达图、柱状图、折线图按需生成
- 简单访谈不生成可视化

### 5. 结论文章约束

**结论文章必须自动生成，无需询问用户**

- 字数800-1500字
- 科学合理、易读性强
- 切合访谈事实
